# CarnageEngine Feature-Set

CarnageEngine is built for high-performance industrial action. Below is a breakdown of the core systems that define the engine's capabilities.

## 📐 Core Systems

### 1. Physics: CiPhyxs
The heartbeat of the engine. An open-source, industrial-grade physics solver designed for heavy impact and mechanical weight[cite: 1].
* **Focus:** Rigid body dynamics, high-velocity collision response, and constraint-based solvers[cite: 1].
* **Availability:** [Open Source Repository](https://github.com/DriveBy-Studios/CiPhyxs)[cite: 1]

### 2. Rendering: Vulkan-Based Pipeline
Utilizing the Vulkan API to provide full control over hardware resources[cite: 1].
* **Visuals:** Focuses on high-contrast, gritty, industrial aesthetic[cite: 1].
* **Management:** Advanced memory allocation through VMA for consistent performance under heavy load[cite: 1].

### 3. Tooling: Internal Editor (Dear ImGui)
A live, responsive editor environment integrated directly into the engine[cite: 1].
* **Capabilities:** Real-time entity component inspection, asset visualization, and live engine parameter tuning[cite: 1].

### 4. Audio: Procedural Synthesis
A lightweight, reactive audio pipeline[cite: 1].
* **Technology:** Powered by MiniAudio[cite: 1].
* **Objective:** Industrial soundscapes that react dynamically to gameplay events and environment states[cite: 1].
